# 🐉 프로젝트: 드래곤의 둥지 (Java Ver.) 구현 계획

주인님, '둥지 짓는 드래곤' 스타일의 던전 경영 시뮬레이션 게임을 자바(Java)로 만들기 위한 설계도입니다.
이 게임은 **'던전 건설'**과 **'침입자 방어'**가 핵심 재미 요소랍니다.

## 1. 기술 스택 (Tech Stack)
- **언어**: Java 17+
- **엔진**: LibGDX (추천) 또는 JavaFX
  - *이유*: 2D 그래픽, 마우스 입력, UI 처리를 깔끔하게 구현하기 위해 게임 라이브러리를 사용하는 것이 좋아요.
- **데이터 저장**: JSON (Gson 라이브러리 사용)

## 2. 핵심 클래스 설계 (Class Design)

### A. 둥지 (Dungeon) 시스템
- **`DungeonMap`**: 맵 전체를 관리하는 격자(Grid) 시스템입니다.
- **`Room`**: 둥지를 구성하는 방입니다.
  - `TrapRoom`: 함정이 설치된 방 (데미지, 디버프)
  - `MonsterRoom`: 몬스터가 대기하는 방 (전투 발생)
  - `ResourceRoom`: 자원(마력, 돈)을 생산하는 방

### B. 유닛 (Unit) 시스템
모든 캐릭터의 기본이 되는 부모 클래스입니다.
- **`Unit`** (공통)
  - 속성: HP, 공격력, 방어력, 속도
- **`Monster` (아군)**
  - `AttackMotion` 클래스를 사용하여 공격 애니메이션 처리
  - 배치 비용(Cost) 존재
- **`Adventurer` (적군)**
  - 침입 목표 (보물 강탈, 드래곤 처치 등)
  - 공포도(Fear) 수치: 너무 높으면 도망감

### C. 게임 진행 (Game Loop)
1. **건설 페이즈 (Build Phase)**
   - 플레이어가 소지금으로 방을 확장하거나 몬스터/함정을 배치합니다.
2. **침입 페이즈 (Invasion Phase)**
   - 모험가들이 입구로 들어와 둥지 깊숙한 곳으로 이동합니다.
   - 각 방에서 전투나 함정 발동이 실시간(또는 턴제)으로 처리됩니다.
3. **결과 페이즈 (Result Phase)**
   - 방어 성공/실패 여부 정산
   - 획득한 자원 및 포로(?) 처리

## 3. 개발 단계 (Milestones)
- **1단계**: 맵을 띄우고 빈 방을 클릭하여 '함정'을 설치하는 기능 구현
- **2단계**: 빨간 점(적)이 들어와서 함정을 밟으면 HP가 깎이는 로직 구현
- **3단계**: 몬스터 배치 및 간단한 자동 전투 구현
- **4단계**: UI 꾸미기 (상냥한 메이드 스타일!)
